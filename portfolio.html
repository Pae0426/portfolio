Recommendations of Note

Description

ブログサイト「note」の記事をスクレイピングで抽出し、形態素解析や評価式を用いておすすめ記事を判別して表示します。

記事の単語・いいね数・長さを主な評価基準としています。 記事内の単語は特定のキーワードで検索された数百件分の記事を形態素解析して頻出単語表を作成し、その表をもとに評価されます。
以下は評価の工夫の一例です。

・長文の記事は極端に高い評価を得やすいので、単語評価の際に記事内のそう単語数との比を取る
・「新着」カテゴリーの記事は比較的いいね数が少ないので、いいね評価を高く設定する
・「人」や「月」といったどの記事にも普遍的に使用されている単語は評価の対象から外す

Voice Kitを用いた模擬面接による、
適正スピーチ速度判定システム

Description

現在履修している授業で友人と共に開発している面接練習システムです。
私は主にコーディングを担当しており、友人はVoice Kitやブレッドボードの配線を担当しています。
また、機能の提案やライブラリの解釈、バグの解析などは試行錯誤を重ねながら協働で進めています。

画像手前の白いボタンを押すと音声が流れ、自己PR・志望動機・学生時代に力を入れたことの中からランダムでユーザーに問いかけます。
ユーザーが答えている間は画像中央のLEDライトで適切な速度で話せているかをリアルタイムで判定します。話す速度が早すぎる場合は赤色、適切な場合は緑色、遅すぎる場合は青色のLEDライトがそれぞれ光ります。
